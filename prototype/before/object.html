«До»: карточка London server 4

Текущая страница объекта в RackTables. Все важные действия (правка, добавление порта, привязка IP) спрятаны под отдельные вкладки и формы — это превращает рутину в 4–5 переходов.

  1. 1. 9 вкладок без иерархии важности. View / Properties / Log / Rackspace / Ports / IP / NATv4 / Tags / Files / 802.1Q — пользователю надо заранее знать, где живёт нужное поле.
  2. 2. Баннер «Has problems» — без причины и без CTA. Это критично: пользователь видит проблему, но не понимает, что с ней делать.
  3. 3. Ports & links — read-only на странице View. Чтобы поменять label порта или добавить cable ID, надо идти на отдельную вкладку Ports → форму → Save → возврат (4–5 кликов на тривиальную правку).
  4. 4. IP addresses показывают связи, но без действий. Привязать новый IP / поменять интерфейс — снова через отдельную вкладку.
  5. 5. Rackspace allocation полезен, но «оторван». Из стойки нельзя одним кликом перейти в карточку соседа или его порт. Соседние строки не кликабельны.
